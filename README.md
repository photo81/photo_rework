# PHOTO사업부 재작업 현황 대시보드

## 파일 구조

```
📁 레포지토리 (GitHub Pages 루트)
├── index.html   ← 대시보드 (수정 불필요)
├── data.xlsx    ← 데이터 파일 (이것만 업데이트)
└── README.md
```

## 데이터 업데이트 방법

1. 로컬에서 `data.xlsx` 수정
2. GitHub에 같은 이름(`data.xlsx`)으로 덮어쓰기 업로드
3. 대시보드 페이지 새로고침 → 자동 반영

## xlsx 파일 필수 시트 구조

### 시트명: `재작업상세`
| 컬럼 | 설명 |
|------|------|
| 구분 | 제휴사 그룹 |
| 제휴사명 | 제휴사 이름 |
| 상품구분 | 포토북 / 캘린더 등 |
| 비용 | 재작업 비용 (원) |
| 주관부서 | 담당 부서명 |
| 사유 | 재작업 사유 |
| 년도 | 주문 연도 (예: 2026) |
| 월 | 주문 월 (예: 4) |

### 시트명: `주문정보`
| 컬럼 | 설명 |
|------|------|
| 제휴사명 | 제휴사 이름 |
| FY | 회계연도 (예: 2026) |
| MO | 실제 달력 월 (1~12) |
| 상품종류 | 상품 구분 |
| 건수 | 주문 건수 |

## 회계연도 기준

- **FY2026** = 2025년 4월 ~ 2026년 3월
- 4~12월: 해당 연도 FY / 1~3월: 다음 연도 FY로 자동 집계

## GitHub Pages 설정

1. 레포지토리 → Settings → Pages
2. Source: `main` 브랜치 / `/ (root)` 선택
3. 저장 후 발급된 URL로 접속
